## 01 BUS — GREEN BUS

รายงานจากระบบ GIS แยกเป็น 6 report ต่อเดือน (ม.ค.–พ.ค. รวม 30 ไฟล์) — สเกลใหญ่มาก:
เดือน พ.ค. เดือนเดียวมี **~99,170 ตั๋ว** ช่องทางหลักคือ Fair Fair B2B 48%, Fair Fair B2C 19%,
ตัวแทนหักวงเงิน 16%, 12Go API 6%

## 02 BUS — FAIR FAIR

ขายตั๋วรายใบเหมือน Green Bus (พ.ค. **66,541 แถว**) แต่เพิ่ม: ประเภทตั๋ว (ซื้อ/เลื่อน/ระบุวัน),
ช่องทาง APP/WEB, และ **คอลัมน์ประกันเสริม CO01TH/CO02TH/CO04TH พร้อมราคา** — เห็นชัดว่าตั๋วไหน
ซื้อประกันพ่วง (พ.ค. 2,420 จาก 66,541 ≈ **3.6% uptake**)

## 03 Cargo

- **CTC คาร์โก้** (ม.ค.–พ.ค. **79,477 ชิ้น**) — พัสดุ 1 ชิ้น = 1 แถว: ผู้ส่ง/ผู้รับ + ที่อยู่เต็ม +
  เบอร์โทร, หมวดสินค้า (free text + taxonomy), น้ำหนัก, จำนวน, ราคา, โปรโมชั่น/key account,
  เที่ยวรถที่ฝากไป
- **FairFair คาร์โก้** (ม.ค.–24 ก.พ. **31,550 ชิ้น**) — มีประเภทบริการจัดส่ง: หน้าสถานี (ส่วนใหญ่),
  จัดส่งถึงบ้าน 399, รับเองที่บริษัท 113, **Delivery เหมาลำ 49 รายการ** (เช่น เหมา 320 ชิ้น
  13,500 บาท ให้ DHL แม่สอด→บางปะอิน)
- **CTC DL & DHL** (ม.ค.–พ.ค. **2,108 แถว**) — last-mile delivery: ที่อยู่ผู้รับเต็ม, น้ำหนัก,
  ส่วนลด, ค่าจัดส่ง
- **Packing** — ขายกล่อง/ซองรายชิ้น: ขนาดกล่อง (G5/G6/G7), ราคา, VAT, สถานีขาย

## 04 Flight

- **FlightReport (BO)** — การจอง 1 รายการ: Transaction ID, PNR, Booking Status, วันจอง vs
  วันเดินทาง, **Advance purchase day (มีให้แล้ว ไม่ต้องคำนวณ)**, Payment Status — ทั้งไฟล์มี
  **215 การจอง** (ทุกรายการ Ticketed), booking window จริง **0–294 วัน, median 18 วัน**
- **GA4.Data.2M.later.csv** — GA4 Explore export: Destination × Travel Date × Event count
  (การค้นหาเที่ยวบิน 18 มี.ค.–18 พ.ค.) ปลายทางยอดนิยม BKK/CNX/DMK
- **Note Flight.pdf** — อีเมลภายใน 17/6/69 ระบุโจทย์โครงการ Data Analysis (AI):
  - วิเคราะห์ **Booking Window, User Intent, Comparative New Users**
  - ทำ **Conversion Funnel Dashboard บน Looker Studio** (ดู CPC/CPA ไม่ใช่แค่ยอดจอง)
  - เตรียม **Data Infrastructure บน BigQuery** รองรับ CDP/CRM
  - ใช้ **Vertex AI / BigQuery ML** ทำ Predictive Analytics, ต่อ GA4 กับ **Meta Pixel**
    ทำ Lookalike Audience
  - ติดตั้ง GTM + Event Tracking แล้ว, แผนยกระดับเป็น Server-Side Tracking / Consent Mode V2

## 05 Insurance

ผู้โดยสารที่ซื้อประกัน 1 คน = 1 แถว แยกไฟล์ตามค่าเบี้ย โครงสร้างเหมือนกันทั้ง 3 ไฟล์
(ม.ค.–พ.ค.): **เบี้ย 24 บาท = 1,379 คน, 38 บาท = 13,252 คน, 45 บาท = 5,948 คน** —
คอลัมน์: ชื่อ, เพศ, **เลขบัตร ปชช.**, อีเมล, เที่ยว/ที่นั่ง, วันเดินทาง

## 06 รถร่วม TPT (Manual)

- **รายได้ค่าคิว** — ใบเสร็จรายใบ: สถานี, หมายเลขรถ, จำนวนเงิน, เจ้าของรถ, สถานะส่งเงิน
- **แผนต่อสัญญารถร่วมปี 2569** — workbook 19 sheets — เป็นไฟล์แผนงาน ไม่ใช่ transaction data

## 07 รถเช่า Fair Car (Manual)

Workbook ทำมือ 16 sheets ต่อเดือน: utilization รายคัน (วันเช่า, ACT/วัน, % เทียบเป้า),
รายได้เทียบเป้ารายเดือน, ราคาตั้ง vs ราคาขายจริง, ประวัติซ่อม, GPS, รายชื่อลูกค้า —
โครงสร้างไม่ standard ต้อง parse เป็นราย sheet

## 08 Fair Super Charge (Manual)

สรุปรายเดือน: ราคาขายไฟ (บาท/kWh), รายได้, หน่วยไฟ (kWh) แยก 3 กลุ่ม — Fleet Greenbus
(6 บาท), Fleet Faircar (6.9), ลูกค้าทั่วไป (6.9) เช่น ม.ค. GB ใช้ 106,173 kWh = 637,039 บาท

# ข้อสังเกตด้านฟอร์แมตไฟล์ (สำคัญตอนทำ ETL)

- ไฟล์ `.xls` ทั้งหมดของ Green Bus **ไม่ใช่ Excel จริง — เป็น HTML table** ต้องอ่านด้วย
  `pandas.read_html()` (ใช้ `read_excel()` จะ error "format cannot be determined")

  และแถวแรกเป็น Grand total ต้องตัดทิ้ง
- แถวแรกของตาราง HTML มักซ้ำ header อีกรอบ → drop duplicate header row
- ไฟล์ manual (Fair Car, แผนต่อสัญญา, FSC) มี merged cells / หลาย header row /
  เลขปี พ.ศ.-ค.ศ. ปนกัน (2569/2026) — parse ยากสุด ควรทำทีหลัง
- วันที่มีหลายฟอร์แมต: `2026-05-01`, `31/05/2026`, `2569-05-31` — ต้อง normalize
