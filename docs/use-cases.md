## NLP (ทำได้จริง แต่ขอบเขตจำกัด)

- **จัดหมวดสินค้าคาร์โก้จาก free text** — ชนิดสินค้าเป็นข้อความอิสระ เช่น "หนังหมู/หนังวัว",
  "ของที่ระลึก,เซรามิก" → Thai text classifier map เข้า taxonomy ที่มีอยู่แล้ว เพื่อความสม่ำเสมอ
  ของการคิดราคาและคัดกรองสินค้าต้องห้าม
- **Parse ที่อยู่ผู้รับภาษาไทย** — บ้านเลขที่/ตำบล/อำเภอ/จังหวัด → แยก field + geocode →
  จัดโซนจัดส่งสำหรับ delivery/DHL
- **Entity resolution ข้ามระบบ** — ลูกค้าคนเดียวกันสะกดชื่อต่างกันในไฟล์บัส/คาร์โก้/ประกัน →
  fuzzy matching ชื่อไทย + เบอร์โทร + เลขบัตร → single customer view (รากฐานของ CDP)
- ⚠️ **Sentiment Analysis** ที่ PDF อยากได้ — **ชุดข้อมูลนี้ไม่มีข้อความรีวิว/ฟีดแบ็กเลย**
  ต้องดึงจากแชท (LINE/FB) หรือรีวิวโซเชียลมาเพิ่มก่อน

## Data Engineering (prerequisite ของทุกอย่างข้างบน)

- ETL รองรับ 4 ฟอร์แมต (xlsx / HTML-in-.xls / GA4 CSV / manual workbook) → BigQuery
  เป็น star schema: `fact_ticket`, `fact_cargo`, `fact_flight_booking`, `fact_queue_fee`,
  `dim_route`, `dim_station`, `dim_vehicle`
- ใช้ convention scheduled query เดิมของทีม: `init_{name}` (full load ครั้งแรก) +
  `tf_{name}` (incremental รายวัน 07:00 ไทย)
- ⚠️ **PDPA:** ไฟล์ตั๋ว/คาร์โก้/ประกัน มี PII ครบชุด — ชื่อ-นามสกุล, เลขบัตร ปชช./พาสปอร์ต,
  เบอร์โทร, อีเมล, ที่อยู่บ้าน — **ต้อง mask/hash (เช่น SHA-256 + salt) ก่อนโหลดเข้า warehouse
  หรือแสดงบน dashboard** ทุกกรณี เก็บ raw PII แยกไว้ในตารางที่จำกัดสิทธิ์เท่านั้น

## Dashboard (quick wins — ข้อมูลพร้อมแล้ว ไม่ต้องรอโมเดล)

1. **ภาพรวมรายได้กลุ่มธุรกิจ** — การ์ดต่อ BU (บัส/คาร์โก้/เที่ยวบิน/รถเช่า/EV) เทรนด์รายเดือน
   เทียบเป้า — เป้ามีอยู่แล้วในไฟล์ Fair Car (`เป้า 2026`) และ FSC
2. **Heatmap เส้นทาง × รอบเวลา × Load Factor** — ชี้รอบที่ว่างเรื้อรัง (ตัด/ควบรอบ)
   และรอบที่เต็มตลอด (เพิ่มรถ)
3. **Flight Conversion Funnel** (Looker Studio — PDF ขอไว้ตรงๆ) — GA4 search → booking →
   paid, กระจาย booking window, top OD pairs (BKK/CNX/DMK), เชื่อมมิติแคมเปญเพื่อดู CPC/CPA
4. **Cargo operations** — ปริมาณ/น้ำหนักรายเส้นทาง, สัดส่วน key account,
   ประเภทจัดส่ง (หน้าสถานี vs delivery vs DHL vs เหมาลำ)
5. **Fleet utilization รถเช่า** — วันเช่าต่อคัน vs เป้า (สูตร ACT/วัน และ % มีใน Excel แล้ว)

# จุดเริ่มที่แนะนำ

เริ่มจาก **flight booking + GA4 → BigQuery → booking window / funnel dashboard** เพราะ:

1. เป็นชุดข้อมูลเล็กที่สุด (215 การจอง ~40 รายการ/เดือน) — ทำจบเร็ว เห็นผลไว
   และตัวเลขที่น้อยนี้เองก็เป็น insight: search จาก GA4 เยอะแต่ conversion ต่ำ →
   funnel dashboard จะชี้จุดรั่วได้ทันที
2. ผู้บริหารระบุความต้องการไว้ชัดเจนที่สุดใน Note Flight.pdf (มี stakeholder รอใช้จริง)
3. ได้วางแพทเทิร์น ETL + PDPA masking ที่จะ reuse กับข้อมูลบัส/คาร์โก้ที่ใหญ่กว่ามากในเฟสถัดไป
